This page explains how to authenticate to the TagoDeploy API and which base URL to use. All HTTP requests must include a valid token in the headers.

#### How to create a token

To create a api token, you need to access the **Tokens** tab in the project **Settings**. Click on the **Create Token** button and follow the instructions to generate a new token.

#### Authentication

TagoDeploy uses token-based authentication. Include a valid token in your HTTP headers for your requests.

You should add the token to the request header using the **"token"** header as documented in the API overview.

```bash
    curl -H "token: <your-token>" \
    https://deploy.tago.io/api/...
```

---


### API Endpoint

TagoDeploy API is available at:

- **Base URL**: `https://deploy.tago.io/api/`

Use the following pattern to construct your endpoint URLs:
```
https://deploy.tago.io/api/<path>
```
